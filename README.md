# Bulk RNA-seq Analysis Pipeline

A SLURM-based pipeline for processing paired-end bulk RNA-seq data: quality control, adapter trimming, alignment, deduplication, coverage track generation, and gene-level read counting.

## Pipeline Overview

```
Raw FASTQ
   │
   ▼
FastQC (raw reads QC)
   │
   ▼
MultiQC (aggregate raw QC report)
   │
   ▼
fastp (adapter/quality trimming, paired-end)
   │
   ▼
FastQC (trimmed reads QC)
   │
   ▼
MultiQC (aggregate trimmed QC report)
   │
   ▼
STAR (genome index + alignment)
   │
   ▼
Picard MarkDuplicates (remove PCR duplicates)
   │
   ▼
samtools index + deepTools bamCoverage (bigwig tracks)
   │
   ▼
featureCounts (gene-level count matrix)
```

## Usage

Submit the pipeline as a SLURM job:

```bash
sbatch run_pipeline.sh
```

### SLURM resource requests
- Time: 1 day
- CPUs: 8
- Memory: 120 GB

Adjust these in the `#SBATCH` header based on sample number/size.

## Steps in Detail

1. **Raw QC** — FastQC run on all raw `*.fastq.gz` files, followed by MultiQC to aggregate into `multiqc_report_raw_reads.html`.
2. **Trimming** — fastp trims adapters and low-quality bases from paired-end reads (auto-detects adapters, min length 50 bp), producing `*_val_1.fq.gz` / `*_val_2.fq.gz` and per-sample HTML reports.
3. **Trimmed QC** — FastQC + MultiQC repeated on trimmed reads (`multiqc_report_trimmed_reads.html`).
4. **Alignment** — STAR builds a genome index (if not already present) and aligns trimmed reads, outputting coordinate-sorted BAMs to `STAR_output/`.
5. **Deduplication** — Picard `MarkDuplicates` removes PCR/optical duplicates from each BAM.
6. **Coverage tracks** — BAMs are indexed with samtools, then converted to bigwig format with deepTools `bamCoverage` for genome browser visualization.
7. **Quantification** — featureCounts generates a gene-level count matrix (`count.out`) from all deduplicated BAMs, using paired-end read counting.

## Output

- `multiqc_report_raw_reads.html`, `multiqc_report_trimmed_reads.html` — QC summaries
- `STAR_output/*_dedup.bam` — deduplicated, sorted, indexed alignments
- `deeptools_output/*.bw` — bigwig coverage tracks
- `count.out` — gene-level read count matrix for downstream differential expression analysis (e.g., DESeq2, edgeR)

## Notes

- Raw and intermediate data files (FASTQ, BAM) are not tracked in this repository — see `.gitignore`.
- Downstream differential expression analysis is not included in this pipeline script.
